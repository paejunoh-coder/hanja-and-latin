# PRD: 한자·라틴어·사자성어 학습 웹 (가칭: WordRoots)

## 1. 개요

### 1.1 제품 한 줄 정의
한자·라틴어 단어와 사자성어를 **직접 손으로 쓰면서**, **예문 속에서 활용하며**, **테마별 시험으로 점검**하는 깔끔하고 심플한 웹 학습 서비스.

### 1.2 배경 / 문제
- 한자와 라틴어는 어원·구조를 이해하면 폭발적으로 어휘력이 늘지만, 기존 학습 도구는 단순 암기(플래시카드) 위주다.
- 학습자는 단어를 "보기"만 하고 끝나, 실제로 **쓰고**, **문장에 적용**하는 능동적 학습이 부족하다.
- 어려운 설명 탓에 진입 장벽이 높다. → **초등학생도 이해할 수 있는 쉬운 설명**이 필요.

### 1.3 목표 (Goals)
- 단어를 보며 연습장에 쓰는 학습 흐름 제공
- 학습 단어를 쓰는 예문 + 빈칸 채우기 연습 제공
- 테마별 단어 묶음 + 테마별 시험 제공
- 모든 단어에 쉬운 배경 설명 제공
- 난이도 순서대로 학습 경로 제공

### 1.4 비목표 (Non-Goals, v1 제외)
- 로그인 / 계정 기능 (진행 상황은 브라우저 `localStorage`에 저장)
- 실시간 음성 인식 / 발음 평가
- 다국어 UI(한국어 UI만 우선)
- 소셜 기능(친구, 랭킹 공유)
- 모바일 네이티브 앱(반응형 웹으로 대체)
- AI 자동 예문 생성(v1은 직접 큐레이션한 데이터 사용)

### 1.5 v1 확정 결정사항
- **로그인 없음**: 모든 학습 진행/시험 결과는 `localStorage`에 저장
- **한자·라틴어·사자성어 동시 출시**: 세 카테고리 모두 v1 범위
- **쓰기 연습 = 따라쓰기 가이드**: 흐린 글자 위에 덧쓰는 방식
- **초기 단어 데이터 = 직접 큐레이션**: 외부 데이터셋 미사용, 수작업 작성
- **사자성어**: 한자 4글자 관용구를 별도 카테고리로 제공. 글자 풀이 + 유래(고사) 설명 포함

## 2. 사용자

### 2.1 타겟 사용자
- 한자/라틴어를 처음 시작하는 초·중등 학생 및 입문 성인 학습자
- 어원 기반으로 어휘력을 키우고 싶은 자기주도 학습자

### 2.2 핵심 사용자 시나리오
1. 사용자는 "가족" 테마를 선택한다.
2. 난이도가 낮은 단어부터 카드로 본다 — 단어, 뜻, 쉬운 배경 설명을 읽는다.
3. 화면을 보며 연습장(종이 또는 화면 쓰기 영역)에 직접 써본다.
4. 단어가 쓰인 예문을 읽고, 빈칸 채우기 문제를 푼다.
5. 테마 학습이 끝나면 테마 시험을 보고 점수를 확인한다.

## 3. 기능 요구사항

### 3.1 단어 학습 (Flashcard / 쓰기 연습)
- **단어 카드 표시**: 단어(한자/라틴어 원문), 발음/독음, 뜻
- **쓰기 연습 영역 (따라쓰기 가이드)**:
  - 흐린(연한 회색) 글자를 가이드로 표시하고 그 위에 덧써서 연습
  - 캔버스에 마우스/터치 펜으로 입력, 지우기 버튼 제공
  - 가이드 보기/숨기기 토글(가이드 없이 스스로 써보기)
- **카드 넘기기**: 이전/다음, 뒤집어서 뜻 확인
- 학습 완료 체크(아는 단어/모르는 단어 표시, `localStorage` 저장)

### 3.2 배경 설명
- 각 단어마다 어원/유래/구성 요소를 **초등학생 눈높이**로 설명
- 예: 한자는 부수·구성, 라틴어는 어근/접두사 풀이
- **사자성어**: 글자별 한자 풀이 + 유래(고사) 이야기를 쉽게 풀어 설명
- 비슷한 단어, 파생 단어 연결(선택)

### 3.2b 사자성어 학습 (idiom)
- **카드 표시**: 4글자 한자, 한글 독음, 뜻
- **글자 풀이**: 네 글자를 한 자씩 풀어서 표시 (예: 馬耳東風 → 말·귀·동녘·바람)
- **유래(고사)**: 사자성어가 생긴 이야기를 쉽게 설명
- **쓰기 연습**: 4글자 따라쓰기 가이드
- **예문 / 빈칸 채우기**: 사자성어를 활용한 문장 + Cloze 문제
- **시험**: 뜻 맞히기, 빈칸 채우기, 글자 채우기 등으로 출제

### 3.3 예문 제공
- 단어당 1개 이상의 활용 예문 + 한국어 해석
- 예문은 학습 중인 단어를 반드시 포함

### 3.4 빈칸 채우기 (Cloze)
- 예문에서 학습 단어 위치를 괄호/빈칸으로 처리
- 사용자가 입력 또는 보기 중 선택으로 채움
- 정답/오답 즉시 피드백 + 해설

### 3.5 테마별 분류
- 단어를 테마(예: 가족, 자연, 숫자, 감정, 시간 등)로 묶음
- 테마별 단어 목록 / 진행률 표시

### 3.6 테마별 시험
- 선택한 테마 단어로 시험 출제
- 문제 유형: 뜻 맞히기(객관식), 빈칸 채우기, 단어 쓰기(자가 채점)
- 시험 결과: 점수, 틀린 문제 복습 리스트

### 3.7 난이도 / 학습 경로
- 단어에 난이도 레벨(예: Lv1~Lv5) 부여
- 기본 학습 순서는 난이도 오름차순
- 사용자는 레벨/테마로 필터링 가능

## 4. 데이터 모델 (초안)

### `words` (단어 / 사자성어 공통)
| 필드 | 타입 | 설명 |
|---|---|---|
| id | uuid | PK |
| category | enum | `hanja` / `latin` / `idiom`(사자성어) |
| term | text | 원문 (한자·라틴어 단어 또는 사자성어 4글자) |
| reading | text | 독음/발음 (사자성어는 한글 독음, 예: "마이동풍") |
| meaning | text | 뜻 |
| char_breakdown | text | (사자성어 전용) 글자별 한자 풀이. 예: 馬(말 마)·耳(귀 이)·東(동녘 동)·風(바람 풍) |
| origin | text | (사자성어 전용) 유래/고사 설명 |
| background | text | 쉬운 배경 설명 |
| difficulty | int | 난이도 1~5 |
| theme_id | uuid | FK → themes |

> 사자성어는 `category = 'idiom'`로 구분하며 `char_breakdown`, `origin` 필드를 추가로 사용한다.
> 한자/라틴어 단어는 이 두 필드가 비어 있을 수 있다.

### `themes` (테마)
| 필드 | 타입 | 설명 |
|---|---|---|
| id | uuid | PK |
| category | enum | `hanja` / `latin` / `idiom` |
| name | text | 테마 이름 |
| description | text | 테마 설명 |
| order | int | 정렬 순서 |

### `sentences` (예문)
| 필드 | 타입 | 설명 |
|---|---|---|
| id | uuid | PK |
| word_id | uuid | FK → words |
| text | text | 예문 원문 |
| translation | text | 한국어 해석 |
| cloze_text | text | 빈칸 처리된 예문 (단어 위치 표시) |

### 학습 진행 / 시험 결과 (localStorage)
> v1은 로그인이 없으므로 Supabase에 `user_progress`/`quiz_results` 테이블을 두지 않고
> 아래 구조를 브라우저 `localStorage`에 저장한다.

```jsonc
// key: "wordroots:progress"
{
  "wordStatus": { "<word_id>": "new | learning | known" },
  "lastReviewed": { "<word_id>": "<ISO timestamp>" }
}

// key: "wordroots:quizResults"
[
  { "themeId": "<theme_id>", "score": 80, "total": 10, "createdAt": "<ISO timestamp>" }
]
```

> Supabase는 v1에서 **읽기 전용 콘텐츠(단어·테마·예문)** 제공에만 사용한다.

## 5. 기술 스택 / 아키텍처

| 영역 | 선택 | 비고 |
|---|---|---|
| 프론트엔드 | Next.js (React) + TypeScript | Vercel 배포 최적화 |
| 스타일 | Tailwind CSS | 깔끔·심플 디자인 |
| 데이터베이스 | Supabase (PostgreSQL) | 단어/테마/예문 (읽기 전용 콘텐츠) |
| 인증 | 없음 | v1 미사용. 진행 상황은 `localStorage` |
| 형상관리 | GitHub | 소스 + 데이터 관리 |
| 배포 | Vercel | GitHub 연동 자동 배포 |

### 5.1 배포 흐름
1. 코드 + 시드 데이터를 GitHub 리포지토리에 push
2. Vercel이 GitHub 연동으로 자동 빌드·배포
3. Supabase 프로젝트의 URL/anon key를 Vercel 환경변수로 주입

## 6. 디자인 가이드
- **컨셉**: 깔끔하고 심플 (여백 충분, 군더더기 없는 UI)
- 단어 카드 중심의 큰 타이포그래피 (한자/라틴어 가독성)
- 무채색 베이스 + 포인트 컬러 1~2개
- 한자/라틴어 구분되는 색 또는 탭
- 모바일 반응형 우선

## 7. 화면 구성 (페이지)
1. **홈**: 카테고리 선택(한자 / 라틴어 / 사자성어), 테마 둘러보기, 이어서 학습하기
2. **테마 목록**: 테마 카드 + 진행률
3. **단어 학습**: 카드 + 쓰기 연습 영역
4. **예문 / 빈칸 채우기**: 예문 + Cloze 문제
5. **시험**: 테마 시험 진행 화면
6. **결과**: 점수 + 틀린 문제 복습

## 8. 마일스톤 (제안)
- **M1 — 데이터 & 기본 학습**: 데이터 모델 구축, 시드 데이터, 단어 카드 + 쓰기 연습
- **M2 — 예문 & 빈칸 채우기**: 예문 표시, Cloze 기능
- **M3 — 테마 & 시험**: 테마 분류, 테마별 시험, 결과 화면
- **M4 — 배포 & 다듬기**: Supabase 연동, Vercel 배포, 디자인 마감, 진행률 저장

## 9. 성공 지표 (예시)
- 한 테마를 끝까지 학습 완료하는 비율
- 빈칸 채우기 정답률 향상 추이
- 테마 시험 평균 점수
- 재방문율(이어서 학습)

## 10. 결정 완료 항목
1. ✅ 로그인 미포함 — `localStorage`만으로 진행
2. ✅ 쓰기 연습 = 따라쓰기 가이드(흐린 글자 덧쓰기)
3. ✅ 초기 단어 데이터 = 직접 큐레이션
4. ✅ 한자·라틴어 동시 출시

## 11. 콘텐츠 범위 (확정)
1. ✅ 규모: **카테고리별 5테마 × 10개** (한자 50 + 라틴어 50 + 사자성어 50 = 총 150개)
2. ✅ 라틴어 발음: **고전식(classical)** 표기
3. ✅ 한자 범위: **교육용 기초 한자** 위주
4. ✅ 사자성어: 별도 카테고리로 5테마 × 10개

### 11.1 한자·라틴어 테마 구성 (제안)
공통 테마 5개를 한자·라틴어 양쪽에 동일하게 적용:
| # | 테마 | 한자 예 | 라틴어 예 |
|---|---|---|---|
| 1 | 자연 (Nature) | 山, 水, 火 | silva, aqua, ignis |
| 2 | 가족·사람 (Family/People) | 父, 母, 子 | pater, mater, filius |
| 3 | 숫자 (Numbers) | 一, 二, 三 | unus, duo, tres |
| 4 | 몸 (Body) | 手, 足, 目 | manus, pes, oculus |
| 5 | 시간·하늘 (Time/Sky) | 日, 月, 年 | sol, luna, annus |

### 11.2 사자성어 테마 구성 (제안)
의미 주제별로 5테마:
| # | 테마 | 예시 |
|---|---|---|
| 1 | 노력·끈기 | 우공이산(愚公移山), 마부작침(磨斧作針) |
| 2 | 우정·관계 | 죽마고우(竹馬故友), 관포지교(管鮑之交) |
| 3 | 어리석음·교훈 | 마이동풍(馬耳東風), 우물 안 개구리=정저지와(井底之蛙) |
| 4 | 자연·세상 이치 | 새옹지마(塞翁之馬), 고진감래(苦盡甘來) |
| 5 | 말·지혜 | 일석이조(一石二鳥), 일거양득(一擧兩得) |

> 단어/사자성어 난이도는 테마 내에서 Lv1→Lv5로 분포시켜 학습 순서를 구성한다.
